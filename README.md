# Panorama Camera Selector

## 1. 목적

본 프로젝트는 파노라마 카메라 센서팩 제작을 위해 **카메라/렌즈 조합과 배치 조건을 입력하고, 파노라마 화각·겹침·블라인드 존·외형 간섭 가능성을 확인하는 GUI 기반 설계 보조 앱**입니다.

주요 목표:

* 카메라 HFOV와 렌즈 개수 기반 파노라마 배치 계산
* 최종 파노라마 목표 각도 기준의 카메라 간 각도 자동 계산
* 기준원 반지름 변화에 따른 카메라 위치와 카메라 간 거리 확인
* 분석 거리 기준의 블라인드 존 발생 여부 및 관련 수치 확인
* 카메라 본체 크기와 렌즈 크기 기반 외형 간섭 검토
* 외형 기준 최소 반지름 및 권장 반지름 계산
* 카메라/렌즈 스펙 저장 및 불러오기
* 계산 로직과 GUI 표시 로직 분리

현재는 **PySide6 GUI**, **배치 계산 코어**, **스펙 입력 패널**, **결과 테이블**, **캔버스 표시**, **SQLite 기반 스펙 라이브러리**가 구성된 상태입니다.

```bash

source .venv/bin/activate
pip install PySide6
```


## 3. GUI 사용 흐름

### 3.1 배치 입력

`배치 입력` 영역에서 다음 값을 설정합니다.

| 항목 | 설명 |
| --- | --- |
| 렌즈/카메라 개수 | 사용할 카메라 또는 렌즈 개수 |
| 기준원 반지름 | 카메라 중심이 배치되는 기준 원의 반지름 |
| 분석 거리 | 블라인드 존 확인 기준 거리 |
| 최종 파노라마 목표 각도 | 목표로 하는 전체 파노라마 각도 |
| 카메라 간 각도 직접 설정 | 자동 계산 대신 카메라 중심 간 각도를 직접 입력할지 여부 |
| 카메라 간 각도 | 직접 설정 모드일 때 사용하는 카메라 중심 간 각도 |
| 전체 yaw offset | 전체 배치의 회전 오프셋 |
| 카메라 간 거리 | 기준원 반지름과 카메라 간 각도로 계산되는 chord distance |

기본 의도는 다음과 같습니다.

* `카메라 간 각도 직접 설정`이 꺼져 있으면 최종 파노라마 목표 각도에 맞춰 카메라 간 각도를 자동 계산합니다.
* `카메라 간 각도 직접 설정`이 켜져 있으면 사용자가 입력한 각도를 그대로 사용합니다.
* 360도 목표 배치는 닫힌 원형 파노라마로 처리합니다.

---

### 3.2 카메라/렌즈 스펙 입력

`카메라 외형` 영역에서 카메라 본체 크기를 입력합니다.

| 항목 | 단위 | 설명 |
| --- | --- | --- |
| 카메라 이름 | - | 저장/불러오기용 이름 |
| 카메라 폭 | mm | 외형 간섭 계산에 사용 |
| 카메라 깊이 | mm | 스펙 기록용 |
| 카메라 높이 | mm | 스펙 기록용 |

`렌즈 스펙` 영역에서 렌즈 정보를 입력합니다.

| 항목 | 단위 | 설명 |
| --- | --- | --- |
| 렌즈 이름 | - | 저장/불러오기용 이름 |
| HFOV | degree | 수평 화각, 파노라마 계산의 핵심 값 |
| VFOV | degree | 수직 화각, 현재는 스펙 기록 중심 |
| 렌즈 직경 | mm | 외형 간섭 계산에 사용 |
| 렌즈 길이 | mm | 스펙 기록용 |
| 외형 clearance | mm | 카메라/렌즈 간 최소 여유 간격 |

---

### 3.3 계산 / 갱신

입력값을 변경하면 GUI가 자동으로 다시 계산되며, `계산 / 갱신` 버튼으로도 수동 갱신할 수 있습니다.

계산 결과는 다음 위치에 표시됩니다.

* 캔버스: 카메라 위치, 기준원, FOV, 블라인드 영역 시각화
* 결과 테이블: 최종 파노라마 각도, 오버랩, 블라인드 여부, 카메라 간 거리, 외형 배치 가능 여부 등
* 상태 표시줄: 계산 완료 메시지

---

### 3.4 외형 기준 최소 반지름 적용

`외형 기준 최소 반지름 적용` 버튼을 누르면 현재 카메라/렌즈 크기와 clearance를 기준으로 계산한 권장 반지름을 배치 입력의 기준원 반지름에 적용합니다.

현재 계산 기준:

```text
필요 폭 = max(카메라 폭, 렌즈 직경) + clearance
최소 반지름 = 필요 폭 / (2 * sin(pi / 카메라 개수))
권장 반지름 = 최소 반지름 * 1.1
```

---

### 3.5 라이브러리 사용

`라이브러리` 탭에서 카메라와 렌즈 스펙을 SQLite DB에 저장하고 다시 불러올 수 있습니다.

기본 DB 경로:

```text
~/.panorama_selector/panorama_specs.sqlite3
```

지원 기능:

* DB 위치 선택
* 현재 카메라 저장
* 현재 렌즈 저장
* 카메라 목록 새로고침
* 렌즈 목록 새로고침
* 선택 카메라 불러오기
* 선택 렌즈 불러오기

## 4. 계산 기준

### 4.1 단위

본 프로젝트의 기본 단위는 다음과 같습니다.

| 값 | 단위 |
| --- | --- |
| 길이 | mm |
| 각도 | degree |
| 면적 계산 내부값 | mm² |
| 결과 표시 면적 | m² 변환 가능 |

---

### 4.2 최종 파노라마 각도

최종 파노라마 각도는 단순히 `카메라 개수 × 카메라 간 각도`가 아니라, **양 끝 카메라의 FOV 외곽선 기준 전체 span**으로 계산합니다.

열린 파노라마에서 자동 각도 계산 시 기본 개념은 다음과 같습니다.

```text
카메라 중심 전체 span = 목표 파노라마 각도 - 렌즈 HFOV
카메라 간 각도 = 카메라 중심 전체 span / (카메라 개수 - 1)
최종 파노라마 각도 = 렌즈 HFOV + 카메라 간 각도 * (카메라 개수 - 1)
```

예시:

```text
렌즈 HFOV = 130°
카메라 개수 = 3
목표 파노라마 각도 = 180°

카메라 중심 전체 span = 180 - 130 = 50°
카메라 간 각도 = 50 / (3 - 1) = 25°
최종 파노라마 각도 = 130 + 25 * 2 = 180°
```

따라서 목표 각도가 180°인 경우, 자동 계산 모드에서는 최종 파노라마 각도도 180°가 되는 것이 의도입니다.

---

### 4.3 오버랩 각도

인접 카메라 간 오버랩은 다음 기준으로 계산합니다.

```text
인접 overlap 각도 = max(0, 렌즈 HFOV - 카메라 간 각도)
```

카메라 간 각도가 HFOV보다 작으면 FOV가 겹치고, 카메라 간 각도가 HFOV보다 크면 블라인드 존이 발생합니다.

---

### 4.4 블라인드 존

블라인드 존은 다음 기준으로 계산합니다.

```text
인접 blind angle = max(0, 카메라 간 각도 - 렌즈 HFOV)
```

분석 거리 기준 arc length와 sector area는 다음과 같이 계산할 수 있습니다.

```text
blind_rad = radians(blind_angle)
blind_arc_length = analysis_distance * blind_rad
blind_sector_area = 0.5 * analysis_distance^2 * blind_rad
```

다만 최근 개발 방향에서는 블라인드 존의 상세 각도/호 길이/면적 값은 결과 테이블에서 제거하고, 사용자가 필요한 경우 다시 표시하는 방향으로 정리할 예정입니다.

---

### 4.5 카메라 간 거리

카메라 간 거리는 기준원 위 두 카메라 중심 사이의 chord distance로 계산합니다.

```text
camera_spacing = 2 * radius * sin(camera_spacing_angle / 2)
```

따라서 기준원 반지름이 커지면 같은 각도에서도 카메라 간 거리가 증가합니다.

## 6. 폴더 및 파일 역할

### 6.1 main.py

GUI 실행용 최상위 진입점입니다.

예상 역할:

```python
from panorama_selector.gui.main_window import main

if __name__ == "__main__":
    main()
```

현재 업로드된 코드 기준으로는 `gui/main_window.py` 내부에도 `main()` 함수가 포함되어 있어 모듈 직접 실행이 가능합니다.

---

### 6.2 core/

계산 로직을 담당합니다. GUI와 분리되어 있어 추후 CLI, 테스트 코드, 리포트 생성에서도 재사용할 수 있습니다.

#### models.py

데이터 모델 정의 파일입니다.

주요 모델:

* `CameraSpec`
* `LensSpec`
* `LayoutConfig`
* `Placement`

`LayoutConfig`에는 카메라 개수, 기준원 반지름, 목표 파노라마 각도, 분석 거리, 카메라 간 각도 직접 설정 여부, yaw offset 등이 포함됩니다.

#### geometry.py

파노라마 배치와 각도 계산을 담당합니다.

주요 기능:

* 각도 정규화
* 360도 닫힌 파노라마 여부 판단
* 인접 gap 개수 계산
* 적용 카메라 간 각도 계산
* 최종 파노라마 각도 계산
* 기준원 위 카메라 위치 계산
* FOV 좌우 edge 각도 계산
* chord distance 계산

#### metrics.py

배치 결과의 수치 지표를 계산합니다.

주요 계산값:

* 카메라 개수
* 목표 파노라마 각도
* 최종 파노라마 각도
* 카메라 간 각도
* HFOV
* 인접 overlap 각도
* 블라인드 존 여부/각도/개수
* 블라인드 arc length
* 블라인드 sector area
* 카메라 간 거리

#### physical_fit.py

카메라/렌즈 외형 간섭 가능성을 계산합니다.

주요 계산값:

* 필요 clearance 폭
* 현재 카메라 간 거리
* 외형 여유 margin
* 외형 배치 가능 여부
* 외형 기준 최소 반지름

#### solver.py

최소 반지름 계산 결과를 바탕으로 권장 반지름을 제공합니다.

현재는 단순 계산 방식입니다.

```text
recommended_radius = min_radius * 1.1
```

## 8. 현재 개발 메모 / 수정 예정 사항

최근 대화 기준으로 다음 사항은 수정 또는 정리가 필요합니다.

### 8.1 캔버스 스케일 기준 정리

요구사항:

* 분석 거리는 화면 스케일을 잡는 기준으로 고정
* 기준원 반지름이 변하면 실제 배치 크기 변화가 보이도록 표시
* 반지름이 커져서 화면 밖으로 나가는 것은 허용
* 기준원 반지름이 자동으로 무시되거나 재스케일되어 보이지 않도록 수정 필요

수정 대상:

```text
gui/canvas_view.py
```

---

### 8.2 결과 테이블 항목 정리

최근 요청에 따라 아래 값들은 결과 레이아웃에서 제거하는 방향입니다.

* 인접 blind angle
* 전체 blind angle
* blind arc length / gap
* blind sector area / gap
* 전체 blind sector area

블라인드 존은 직접 화면에 거리로 표시하기보다, 결과 레이아웃에서 필요한 요약값만 보이도록 정리합니다.

수정 대상:

```text
gui/main_window.py
core/metrics.py  # 계산 자체를 유지할지, 표시만 제거할지 결정 필요
```

---

### 8.3 카메라 간 거리 표시 로직 확인 필요

현재 GUI 표시용 카메라 간 거리 계산과 core 배치 계산의 카메라 간 각도 기준이 다를 수 있습니다.

의도한 자동 계산 기준:

```text
카메라 간 각도 = (목표 파노라마 각도 - HFOV) / (카메라 개수 - 1)
```

반면, 배치 입력 패널의 표시용 계산이 단순히 다음처럼 처리되면 실제 core 계산값과 달라질 수 있습니다.

```text
카메라 간 각도 = 목표 파노라마 각도 / 카메라 개수
```

수정 대상:

```text
gui/layout_panel.py
```

권장 방향:

* `layout_panel.py`에서 직접 각도를 재계산하지 말고 `core.geometry.applied_camera_step_deg()`를 재사용
* 이를 위해 `LayoutPanel`이 현재 렌즈 HFOV를 알 수 없다는 구조적 한계가 있으므로, 표시용 카메라 간 거리 갱신을 `MainWindow.calculate()` 결과 기준으로 업데이트하는 방식 검토

---

### 8.4 외형 간섭 계산 기준 확인 필요

외형 간섭 계산에서 카메라 간 거리를 구할 때, 열린 파노라마 배치의 실제 카메라 간 각도를 반영해야 합니다.

권장 방향:

* `physical_fit.py`에서 `chord_distance()` 호출 시 실제 적용 카메라 간 각도를 넘겨주는 방식 검토
* 자동 목표각 모드와 직접각 모드 모두 같은 기준으로 계산되도록 정리

수정 대상:

```text
core/physical_fit.py
```

---

### 8.5 캔버스 caption 중복 문구 정리

현재 caption에 `Step`, `HFOV` 정보가 중복 표시될 수 있으므로 정리하면 좋습니다.

수정 대상:

```text
gui/canvas_view.py
```

## 9. 설계 철학

본 프로젝트는 다음 원칙을 따릅니다.

* 계산 로직과 GUI 표시 로직 분리
* 단위는 기본적으로 mm / degree로 통일
* 파노라마 최종 각도는 양 끝 FOV 외곽선 기준으로 계산
* GUI는 입력/표시 역할에 집중
* core 계산 함수는 추후 테스트/CLI/리포트 생성에서도 재사용 가능해야 함
* 기능 추가 시 기존 코드를 과도하게 리팩토링하지 않고 필요한 파일만 수정
* 센서팩 초기 설계 검토에 필요한 수치와 시각화를 빠르게 확인하는 것을 우선

## 11. 요약

현재 프로젝트는 파노라마 카메라 배치 검토를 위한 핵심 구조가 잡혀 있습니다.

```text
입력 패널
  → core 계산
  → 캔버스 시각화
  → 결과 테이블 표시
  → 스펙 라이브러리 저장/불러오기
```

현재 구조를 유지하면 다음 장점이 있습니다.

* 새로운 계산식이나 출력 기능을 core/exporters에 추가하기 쉬움
* 카메라/렌즈 데이터베이스를 점진적으로 확장 가능
* 센서팩 설계 초기 단계에서 배치 조건을 빠르게 비교 가능
